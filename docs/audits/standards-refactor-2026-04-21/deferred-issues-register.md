# Deferred Issues Register

This register captures problems discovered during standards analysis that are not fully solved by standards compliance alone. These should become separate bugs, test tasks, or design tasks.

| ID | Area | Location | Problem | Suggested Follow-Up |
| --- | --- | --- | --- | --- |
| D01 | Electron security | `electron/src/main.ts` | `api:call` forwards arbitrary method strings and params after trusting TypeScript annotations. | Addressed by RPC method allowlisting, params shape validation, and package-local malformed payload tests. |
| D02 | Electron security | `electron/src/main.ts` | `dialog:openFile` accepts raw dialog options from renderer. | Addressed by dialog option sanitization and package-local tests that reject unsupported fields and malformed filters. |
| D03 | RPC security | `rust/crates/pumas-rpc/src/server.rs` | CORS allows any origin/method/header. | Addressed by loopback-only origin checks, narrowed methods/headers, and unit tests; revisit if a non-loopback web client becomes supported. |
| D04 | RPC lifecycle | `rust/crates/pumas-rpc/src/server.rs` | Spawned Axum server handle is discarded; shutdown/panic handling is not owned. | Addressed by returning an owned `ServerHandle`, logging server task errors, and aborting the task during explicit or drop-based shutdown. |
| D05 | Async lifecycle | `rust/crates/pumas-core/src/api/builder.rs` | Startup recovery tasks were spawned without tracked handles. | Addressed by routing builder-started background tasks through `RuntimeTasks`, aborting them during `PumasApi` shutdown, and adding a task-owner shutdown test. |
| D06 | Python concurrency | `torch-server/model_manager.py` | Slot registry and config state can race across concurrent load/unload/configure calls. | Addressed for slot reservations, unload transitions, and model-limit updates with a manager-level async lock; add route-level concurrent request tests when Python API integration tooling is adopted. |
| D07 | Python composition | `torch-server/serve.py` | `create_app()` mutates a module-global FastAPI app. | Addressed by fresh app construction and duplicate-route tests; keep watching for app-state sharing in future sidecar tests. |
| D08 | LAN exposure | `torch-server/control_api.py` and `serve.py` | LAN mode can bind to `0.0.0.0` without visible auth policy. | Addressed by requiring both `PUMAS_TORCH_ALLOW_LAN=1` and `PUMAS_TORCH_API_TOKEN` for LAN binding, and by enforcing token auth on sidecar API routes when a token is configured. |
| D09 | Test isolation | `rust/crates/pumas-core/tests/api_tests.rs`, `rust/crates/pumas-core/src/tests.rs` | Process-global env/path overrides require serialized guards; broader suite parallel safety needs audit. | Addressed for the guarded API test surface by documenting the isolation contract and adding `scripts/rust/check.sh test-isolation`, which repeats the guarded in-crate API tests and `api_tests` integration binary with multiple test threads. |
| D10 | Generated/runtime state | workspace tree | Runtime directories under `launcher-data/` and `rust/target/` appear locally and can pollute audits. | Addressed by adding `scripts/dev/list-audit-files.sh`, which enumerates tracked and non-ignored audit files while explicitly excluding generated/runtime-heavy paths and retaining tracked plugin manifests. |
| D11 | UX/a11y | `frontend/src/components/InstallDialog.tsx`, `MigrationReportsPanel.tsx` | `window.confirm` bypasses app focus and style system. | Addressed by replacing install cancellation and migration report confirmations with `ConfirmationDialog`; continue using app-owned dialogs for future destructive actions. |
| D12 | Frontend tests | custom role-button/backdrop components | Keyboard/focus tests may be incomplete for custom interactive generic elements. | Addressed for the audited custom controls: app launcher indicator, version selector controls, and import lookup metadata rows use native buttons; sidebar Escape handling no longer uses a generic JSX keyboard handler; `InstallDialog`, `ModelMetadataModal`, `ConflictResolutionDialog`, and `MappingPreviewDialog` use named dialogs with native backdrop buttons and dismissal tests. |
| D13 | Dependency ownership | `electron/package.json`, `frontend/package.json`, root `package.json` | Workspace-local scripts rely on root TypeScript/ESLint tooling. | Addressed for TypeScript/ESLint command ownership by package-local declarations and workspace-scoped verification; continue dependency audits for Rust and release tooling. |
| D14 | Release integrity | release docs/scripts | No visible checksum generation workflow for release artifacts. | Addressed by the release artifact contract and CI checksum step; checksum generation now excludes `checksums-sha256.txt` from its own digest list and fails if no artifacts are staged. |
| D15 | Performance | Rust model index/library/download/conversion paths | No Criterion benchmarks found for performance-sensitive claims. | Add representative benchmarks after decomposition stabilizes APIs. |
| D16 | Release CI | `.github/workflows/build.yml` | Workflow attempted to build and publish `pumas_rustler` artifacts even though release docs exclude BEAM-dependent NIF outputs from default CI. | Addressed by excluding `pumas_rustler` from default Rust build/test commands and removing Rustler artifacts from the release staging script; add a dedicated BEAM-aware job before publishing NIF artifacts. |
| D17 | Rust tooling | `.github/workflows/build.yml`, `scripts/` | Rust fmt, clippy, doc-test, all-features, and no-default-feature checks were not available through one reusable command or CI job. | Addressed by adding `scripts/rust/check.sh` and a Linux Rust quality CI job; add workspace lint policy in a follow-up. |
| D18 | Rust unsafe policy | `rust/Cargo.toml`, `rust/crates/*/Cargo.toml`, `rust/crates/pumas-core/src/platform/process.rs`, `rust/crates/pumas-core/src/platform/paths.rs`, `rust/crates/pumas-core/src/metadata/atomic.rs` | No workspace lint baseline governed unsafe operations while unsafe isolation remains in progress. | Partially addressed by inheriting `unsafe_op_in_unsafe_fn = deny`, documenting current unsafe FFI/raw-pointer blocks with `SAFETY:` notes, centralizing process-alive, process-detach, and Windows display-path FFI, and surfacing Unix fsync failures; follow up by replacing the remaining conversion raw-pointer bridges before ratcheting direct unsafe usage. |
| D19 | Frontend Torch UI | `frontend/src/components/app-panels/sections/TorchModelSlotsSection.tsx` | The library SafeTensors list renders every matching model while the overflow copy says "Showing first 20", so the UI can misrepresent long lists. | Addressed by limiting the rendered SafeTensors preview to the shared 20-model preview limit used by the overflow copy. |
| D20 | Frontend Ollama UI | `frontend/src/components/app-panels/sections/OllamaModelSection.tsx` | The library GGUF list renders every matching model while the overflow copy says "Showing first 20", so the UI can misrepresent long lists. | Addressed by limiting the rendered GGUF preview to the shared 20-model preview limit used by the overflow copy. |
| D21 | Frontend shortcut state | `frontend/src/hooks/useVersionShortcutState.ts` | `toggleShortcuts` restores the previous optimistic state when the API throws, but a non-throwing `{ success: false }` response can leave the optimistic state applied. | Addressed by treating unsuccessful shortcut-toggle responses as API failures, restoring the previous state, and adding explicit coverage for non-throwing backend failures. |
| D22 | Desktop bridge drift | `electron/src/preload.ts`, `electron/src/rpc-method-registry.ts`, `rust/crates/pumas-rpc/src/handlers/mod.rs` | Preload forwarded `install_deps`, `set_version_shortcuts`, `toggle_version_menu`, `toggle_version_desktop`, `launch_version`, `call_plugin_endpoint`, `launch_app`, and `stop_app`, but those method names were not registered in the Electron RPC allowlist or Rust JSON-RPC dispatcher. The new allowlist exposed this as a runtime rejection for any active caller. | Addressed by replacing stale preload aliases with supported RPC calls, adding the missing `call_plugin_endpoint` Rust JSON-RPC handler and Electron schema, and enforcing preload-to-registry coverage with `electron/tests/preload-rpc-contract.test.mjs`. |
